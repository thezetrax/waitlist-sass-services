# UI/UX Case Study

This is a case study of the UI/UX design for a waitlist application.

As a waitlist application, the aim is to provide a seamless and user-friendly 
experience for users to join the waitlist for a product, event or service.
The UI/UX design should be intuitive, easy to navigate, and visually appealing.

## User Identification

- A user tying to setup a waitlist for a product, event or service.
- A user who has wants to get information about a product, event or service.
- A user who has already joined a waitlist for a product, event or service aiming
to get notified about updates.
- Anonymous user who wants to read about a product, event or service.

## User Experience

- Anonymous user can read about a product, event or service.
- Logged-in user can subscribe or unsubscribe from a waitlist.
- Logged-in user can manage their waitlist, update their information, and receive
notifications about updates, and waitlist changes
- Logged-in user can publish a product, event or service.
- Logged-in user can manage their waitlist publication status, publish announcements, and manage their waitlist.
